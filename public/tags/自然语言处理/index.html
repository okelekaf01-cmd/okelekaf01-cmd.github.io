深入解析 OpsMind 三维意图识别系统

一、背景与动机 1.1 问题背景 在企业智能运营场景中，用户查询种类繁多，传统的关键词匹配或规则引擎难以应对： 用户查询示例： ├── "公司的请假流程是怎样的？" → 知识库查询 ├── "分析本季度各产品线的毛利率构成" → 数据分析 ├── "查看新用户注册到付费的转化漏斗" → 数据分析 └── "各部门薪资分布和离群值分析" → 数据分析 1.2 之前方案的局限 方案 局限性 关键词匹配 无法理解语义，“不想要分析"会被误判为"分析” 规则引擎 规则难以穷尽，维护成本高 简单分类 仅区分"查询"和"分析"，无法提供业务上下文 1.3 我们的解决方案 引入 LLM 驱动的三维意图识别系统，实现： ✅ 语义级别的深度理解 ✅ 多维度精细分类 ✅ 业务上下文自动注入 ✅ 图表推荐智能引导 二、系统架构概览 2.1 整体架构图
